THE PROBLEM

AI 写代码很快,
但项目一多,一切开始失控

这是每个用 AI 编程的人都会撞上的墙——瓶颈早已不是"写代码",而是治理。

01项目记忆管不住

聊天记录就是全部项目记忆。会话一关,上下文归零,AI 不知道"上次干到哪"。

02规则越改越乱

"改业务需求"和"改协作规则"混在一起,各项目的规则版本悄悄漂移,谁也说不清哪份是最新。

03全局状态没人看得清

多个项目、多个 AI 会话并行,谁在等谁、哪里阻塞了,要翻遍所有仓库才能拼出答案。

所以我不是做了一个 App,而是给自己造了一套操作系统。

LAYER 1 · WORKFLOW

四个角色,五道门禁

默认是通用助手,不越权、不擅自进流程。一句"你是 PM",AI 切换角色、加载规则、进入门禁。每个角色边界写死,不靠自觉。

LAYER 2 · ORGANIZATION

往上一层:这不是提示词,
是一家"薄公司"的组织架构

单个角色智能体到处都有。差异在于:这套工作流之上,还有一整层组织设计——指挥官—参谋长制,让一个人真正"开公司"般地运营多个 AI 项目。

三条组织法则
① 结构跟着物理走——一个头衔 = 一个目录 + 一个会话
② 参谋不决策——AI 出方案办事务,拍板永远在人
③ 半自动是设计选择——决策权下放留给下一次升级
框架会自我修正(BCR 闭环)
提报→评估→采纳→落地真源→回流全部下游
  • BCR-004/006角色 6 → 4:UI 并入 PM,Tester 并入开发自测
  • BCR-007组织定位升级:指挥官—参谋长制(薄公司)
  • BCR-008回流护栏:同步脚本自检,防误改下游

规则版本用 .workflow-version 游标逐项目追踪,不漂移。

THE CONSOLE · WORKBOARD v0.2

看板,已经长成
「Web 端项目迭代 IDE」

v0.1 是只读聚合看板;v0.2 把生态全局、迭代进度、每一句 AI 对话装进同一个页面。严格只读、不回写任何项目——看得全,但不越权。

01工作台 02项目会话 03跨项目 04部署 05接入诊断
🧭生态分层钻取

打开即是生态根总览:参谋长席位、框架真源与公告板摘要、三个子项目卡片;点卡片钻取进单项目,面包屑一键返回。

🔗角色—会话映射

每个项目的 5 个角色(PM / Architect / Developer / DevOps / General)+ 生态根参谋长席位,各自绑定一条真实开发会话(Claude Code / Codex 双源,带来源徽章);配角色自动识别辅助标注,映射持久化、可随时换绑。

💬对话只读回放

点开任意已映射的角色卡,按时间顺序完整回放该角色的全部对话,Markdown 渲染——PM 怎么写 PRD、Review 怎么打回,一句不落。

🧵双层迭代时间轴

上层:项目迭代总览轴(v0.1 → v0.N);下层:单个迭代的六阶段流水线。已完成的看产出摘要,进行中的看卡在哪;点开阶段抽屉,还能一键跳到「当时的对话」。

♻️生产数据自动同步

生产服务器定时拉取各项目仓库,看板数据始终新鲜;同步异常和数据最后更新时间在接入诊断视图里直接可见,坏了不装死。

v0.1 只读聚合看板→ v0.2 会话回放 + 双层时间轴(已上线)→ v0.3 映射完整管理 + 页面输入

终态规划 · Web Agent Console 统一角色工作台:在网页上选项目、选角色、直接对话,后端在对应项目目录驱动 CLI Agent 并注入角色语义,输出实时回显、权限确认在线完成——把"cd 进目录 + 打开 CLI + 输入你是某角色"收敛成一个页面。
