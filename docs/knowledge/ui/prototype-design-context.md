## 一、协作机制（谁把控什么 —— 先理解这个）

原型图的"把控"是三层，混在一起才乱：

| 层 | 谁 | 做什么 |
|----|----|----|
| ① 设计方案 | **PM** | 出界面方案、写 design 规格、读回验证。BCR-004 已把界面要点并入 PM |
| ② 方向决策 + 评判 | **Owner** | 拍板方向（抽屉？选择即当前？）、判断好不好、提意见 |
| ③ 执行画图 | **design agent**（claude.ai/design 里另一个 agent，不可靠） | 按规格画；**会假完成**（回"改好了"但文件没动），必须 PM 读回验证 |

**设计真源 = PRD 界面要点 + 线框（PM 把控、可靠）；原型图 = 可选可视化辅助**（画不对不强求，实现照 PRD + `frontend/` 现网代码）。**每轮改完必读回验证**——不然一句"改好了"就带过去了，实现阶段才发现原型图和代码对不上。

## 三、生成约定（喂 design agent 的规则）

### 怎么把本文档喂给 design agent
design agent **可直接读取本文档的 GitHub 版本**（`huiyiyouck/niuma-cheng-workboard` 仓 `docs/knowledge/ui/prototype-design-context.md`）——**2026-07-14 实测通过**（design 读到并按它把迭代时间轴一次补对）。**优先让它读 GitHub（免手动贴）、读不到再手动贴全文**；真源在 git，design 只读不写。

### design 沙盒 ≠ frontend 生产（四层不兼容，来自 PRD §7）
原型图是**高保真视觉 + 交互蓝本，不是可部署代码**：沙盒用 `window.React` / design 自建 `lib/*` 与 primitives / `sampleData` 演示数据 / design 运行时——与 `frontend/`（Vite + 标准 import + `lucide-react` + shadcn/ui + 真实 `/api/snapshot`）四层不兼容。**原型图留存作视觉蓝本，实现时 Developer 翻译（保留 Tailwind className + 交互逻辑），后端从零。**

### 给 design agent 的规格怎么写（PM 自用清单）
1. **点名文件**：说清改哪个组件文件（如 `components/shell/WorkboardAppV21.tsx`），别只说"对话页"。
2. **结构 + 视觉 + 交互三要素**：容器/布局、视觉 token、点击/拖拽行为都写全。
3. **sampleData**：新组件要它配演示数据（各状态齐）。
4. **反假完成**：若上一轮声称改了、读回发现文件没变，规格里**明写"上轮实际未改，请这次真正修改该文件并确认内容变了"**。
5. **PM 读回验证**：DesignSync `get_file` 读目标文件，确认规格里的结构真出现，才算做对。

### 对话查看器抽屉细节（核心组件，展开）
- **容器**：右侧滑出抽屉 ~72%（复用 `Drawer`），从项目会话第一级点角色触发。
- **左侧角色菜单**（US-2）：竖排 5 角色（产品/架构/开发/运维/General）各带会话数；点击切角色、当前高亮深色实心；**同时是拖拽放置目标**（拖会话到角色 = 改归属 US-10）。
- **右内容区顶栏**：会话标题 + 项目·消息数 + 来源标签(Claude/Codex) + 迭代标签「所属迭代 X·推断」+ 关闭。
- **当前 vs 历史 + 选择即当前**（US-11）：当前会话标「当前」，下面「历史会话(N)·可回溯」；**点任一会话即成为当前，无「设为当前」按钮**；默认当前=时间最新；**会话行显示 标题/时间/迭代标签/来源标签，不显示角色**（左边已选）。

## 五、关键交互决策（+ 为什么）

- **选择即当前**（US-11）：点会话即成当前、无额外按钮 → 像翻对话历史一样自然；`currentByRole` 前端临时、不持久（刷新回最新）。
- **1:N 归属**（US-3）：一角色可挂多会话（v0.2 是 1:1）；每会话按 `manualRole ‖ detectedRole ‖ General` 归入唯一角色。
- **当前/历史分层**：当前有且仅一个（默认时间最新），其余为历史可回溯。
- **拖拽打标签纠错**（US-10）：识别错 → 拖到别的角色 / ⌄菜单改；`manualRole` 本地持久化（事实纠正），不回写会话源。
- **强限制选择器**（US-3）：空态只列识别为该角色的会话，逼出"识别对不对"，识别错走打标签纠错。
- **迭代标签"尽力而为"**（BCR-013 局限）：一会话未必对应单一迭代，标签是"尽力归属"非绝对真相 → UI 标「推断」、留中性「未归属」态。
- **General 兜底**（US-4）：识别不出的会话归 General，无 Unknown 孤儿。
