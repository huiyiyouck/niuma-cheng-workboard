### 设计 §6 回环 · Developer 实现阶段发现（2026-07-18）

**触发**：实现阶段 R1 搭隔离开发环境（新建 `workboard_dev` 库）时，`applyMigrations` 跑 `001` 报 42601 语法错，暴露迁移引擎缺陷 + 设计 §6 假设瑕疵。**Owner 拍板回设计阶段**由 Architect 订正 §6，R1 暂停待订正（设计门禁待 Architect 自行改「已定稿」→「修改中」，本条为 Developer 发现移交，不代改门禁/设计正文）。

**发现（给 Architect 订正 §6 的依据，证据充分）：**
- **migrations 引擎 DO `$$` 块 bug**：`migrations.js` `splitSqlStatements`（`:100-143`）按 `;` 拆语句、**不识别 dollar-quoted 块**；`001_init.sql` 的 session_mappings 约束用 `DO $$ BEGIN IF NOT EXISTS(...) THEN ALTER TABLE ...; END IF; END $$;`，内部分号被误拆 → 全新库执行 `001` 语法错（42601）。
- **生产为何没暴露**：生产 `workboard` 库 `schema_migrations` **已记 version 1**（历史某时点写入），`applyMigrations` 跳过 `001`，故 bug 只在**全新库/重建库**（如 `workboard_dev`）暴露。
- **设计 §6 假设被证伪**：§6「全新库靠 `001` 建 `session_mappings` → `002` DROP 收口」不成立——全新库 `001` 跑不起来、`applyMigrations` 直接失败，`002` 的 `UPDATE ... FROM session_mappings` 无表可依。
- **生产数据事实**（供订正知情）：生产 `session_mappings` **6 行真实手动映射**，`002` 步骤2 须正确迁进 `manual_role`（不可简单 DROP 丢数据）；`claude_sessions` 现无 `manual_role`（002 未跑），`detected_role`/`source` 已在。

**建议 §6 补充方向**（供 Architect 定，不替其决策）：实现阶段须先让迁移引擎支持 dollar-quoted 块（修 `splitSqlStatements`，或改 `executeMigration` 执行方式），使全新库 `001→002` 链成立；`002` 本身纯 `ALTER/UPDATE/DROP`、不含 DO 块、不受影响。属迁移引擎健全化，是 `002` 可靠落地前提。

**环境已就绪待续**（R1 隔离准备，不丢）：本地 `.env` 已同步生产现行 PG 密码（Owner 授权取自 `/opt/workboard-prod/app/.env`，旧密码 18 字符已失效、现行 19 字符）、新建空库 `workboard_dev`、`.env` 切至 dev 库；生产 `workboard` 零触碰。§6 订正定稿后 R1 从此环境续。

### 设计 §6 回环 · Architect 订正回应（2026-07-18）
