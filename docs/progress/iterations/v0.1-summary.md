# v0.1 迭代归档摘要

## 基本信息
- 迭代目标：交付「项目管理工作台」第一版——本地可运行的**只读**看板，配置驱动聚合牛马程生态各项目（workflow / xiaobao / ai / coordination / workboard）的工作流状态、接入诊断、跨项目协作与待办，给 Owner 一处全局视图。
- 开始时间：2026-06-23
- 关闭时间：2026-06-24
- 最终状态：已完成（生产实测通过、已上线）

## 完成内容
- **四视图**：工作台（统计条 + 跨项目协作横条 + 跨项目待办汇总 + 项目卡网格）/ 部署 / 接入诊断 / 跨项目（需求池 / 谁等谁 / 沟通 / BCR 四 Tab）。
- 项目详情**右抽屉**（迭代 / 角色 / 待办 / 入口 / 配置路径）。
- **配置驱动接入**（`projects.config.json`，按 `kind` 解析与降级）+ 接入校验。
- **60s 自动轮询**，单项目错误隔离。
- 技术栈：React 18 + Vite + Tailwind v4 + shadcn 前端（对齐小报）+ 本地 Node 聚合后端（只读读取 Markdown / config）。
- 部署：测试环境 `115.191.43.79:8088`（systemd `workboard-api-test`）；生产 `workboard.huiyiyou.cloud`（systemd `workboard-api-prod` :5181 + nginx 443 + 独立发布目录），Owner 实测通过。

## 关键决策
- UI / 原型先行（非常规顺序），PRD 据原型回填。
- **定位升级**：跨项目看板 → 项目管理工作台 / 项目管理统一中枢（第一版仍只读）。
- 技术栈从立项「静态 HTML/CSS/JS」改为 **React + shadcn**（对齐小报、复用 Figma 原型）。
- **风格统一全局宗旨**：沿用小报 shadcn token（base 16px、系统 sans、主色 `#030213`）。
- coordination 不进项目网格，提为顶部**协作横条**。
- 跨项目第 4 Tab 加 **BCR**；需求池 / BCR 状态筛选替代「查看全部」。
- 第一版严格**只读**。
