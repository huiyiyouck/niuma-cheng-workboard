## 非常规顺序说明

本迭代不按 `PRD → UI` 常规顺序，采用 **UI / 原型先行**：

- **原因**：Owner 决定以 UI 原型驱动功能定义——先确定界面风格与结构、出原型图，再根据原型图回填 v0.1 功能 PRD。
- **处理**：先跑 UI 方案阶段（`v0.1-ui.md`），PRD 阶段标记「待回填」；待原型定稿后由 PM 创建 `v0.1-prd.md`，再按流水线回到设计 / 实现阶段。
- **依据**：`standard-iteration-quick.md` §7「Spike 可跳阶段但须在迭代记录写原因」。
- **角色说明**：Owner 拟后续合并 / 精简 UI 角色，本迭代 UI 方案由 PM 兼任产出；真正增删角色属工作流基线改动，需单独走 BCR，不在本迭代内执行。
