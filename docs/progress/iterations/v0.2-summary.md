# v0.2 迭代归档摘要

## 基本信息
- 迭代目标：从「只读聚合看板」升级为「Web 端项目迭代 IDE」——新增「项目会话」视图，实现生态分层结构展示、角色会话映射配置、对话内容只读查看、迭代时间轴、生产数据源自动同步
- 开始时间：2026-07-04
- 关闭时间：2026-07-07
- 最终状态：已完成
- 迭代类型：标准迭代（UI / 原型先行变体）

## 完成内容

### 核心功能（新增「项目会话」视图）
1. **生态分层结构展示**：生态根目录视图（参谋长席位 + 框架真源/公告板只读卡片 + 3 个子项目卡片）→ 子项目钻取（面包屑导航 + 5 角色卡片 + 双层时间轴）
2. **角色会话映射配置**：支持 5 角色（PM/Architect/Developer/DevOps/General）+ 参谋长席位映射；三步弹窗（子项目）+ 单步弹窗（参谋长）；SQLite→PostgreSQL 持久化
3. **对话内容只读查看**：按角色卡片钻取对话视图；用户消息左对齐、Claude 消息右对齐；纯文本渲染（Markdown 延后 v0.3）；底部只读提示
4. **迭代时间轴**：双层结构——上层项目迭代总览轴（INDEX.md 版本列表，横向滚动）+ 下层迭代详情轴（vX.Y.md 6 阶段流水线）；进行中显示阶段状态（✅/🔵/⚪/🔴），已完成显示产出内容摘要（三级降级）
5. **Claude Code 会话同步引擎**：本地 JSONL 文件解析 + 增量同步（每 30 秒）；角色自动识别（精确匹配 + 关键词加权）；配置驱动白名单过滤
6. **双数据源同步**（IRC-002）：本机 + 服务器 rsync 镜像，`claude_project_id` 支持数组
7. **Codex 会话源兼容**（IRC-003）：source 列区分 Claude Code / Codex，徽章展示

### 技术升级
1. **数据库从 SQLite 升级到 PostgreSQL**（IRC-001）：服务器已有 PG16，不另装 SQLite；`pg` 驱动 + 连接池；`.env` 环境变量管理；SSH 隧道本地开发
2. **健康检查端点**：`GET /api/health` 返回 `{status, db, version}`
3. **版本号升级**：`0.1.0` → `0.2.0`

### 生产部署
- 生产环境：`workboard.huiyiyou.cloud`
- 后端：systemd `workboard-api-prod`，绑定 `127.0.0.1:5181`
- 数据库：PostgreSQL 16 `workboard` 库
- 前端：`/var/www/workboard.huiyiyou.cloud`

## 关键决策

| 决策 | 背景 | 结论 |
|------|------|------|
| 数据库选型变更 SQLite → PostgreSQL | 服务器已有 PG16，避免另装 SQLite | ✅ 变更加 PostgreSQL（IRC-001） |
| 角色自动识别是否 v0.2 必做 | Reviewer 建议后置降低成本 | ✅ v0.2 必做，作为辅助标签 |
| 生产数据源同步脚本谁写 | Developer 还是 DevOps | ✅ Developer 出方案，DevOps 部署阶段落地 |
| root_session_id 迁移策略 | 自动迁移 vs 手动改 | ✅ 设计阶段统一改 schema，手动删除，不写迁移代码 |
| git 同步策略 | pull vs fetch + reset --hard | ✅ fetch + reset --hard（镜像语义） |
| 产出摘要最低形态 | 详细/一句话/阶段名+链接 | ✅ 三级降级：最低保证（阶段名+角色+日期+链接）、一句话 best-effort、详细不做 |
| PostgreSQL 版本化迁移机制 | 实现 vs YAGNI | ✅ v0.2 已实现（`schema_migrations` 表 + SQL 文件迁移 + 事务保护） |
| 五个角色是否全部映射 | 4 核心角色 vs 5 个含 General | ✅ 五个角色全部映射，无一例外 |
| 会话列表交互 | 服务端搜索分页 vs 前端本地过滤 | ✅ 首屏 50 条 + 前端本地过滤，不做服务端搜索 |
