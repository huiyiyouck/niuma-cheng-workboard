> 用途：在 v0.1 原型基础上，新增「项目会话」视图（第 2 个左菜单项），采用**生态分层结构 + 文档做轴/会话做钻取**：默认显示生态根目录，点击子项目展开迭代时间轴 + 角色卡片。配合 `v0.2-ui.md`（UI 方案）使用。

## 0. 给 Figma AI 的修改总指令

> 这是「项目管理工作台」v0.2，在 v0.1 四视图基础上新增第 5 个视图「项目会话」。**核心改动：采用生态分层结构 + 文档做轴/会话做钻取**。请据此修改：
> 1. **左菜单新增第 2 项「项目会话」**（放在「工作台」之后，按使用频率排序：工作台 / 项目会话 / 跨项目 / 部署 / 接入诊断）。
> 2. **默认显示【生态根目录视图】**，不要直接铺出所有项目。
> 3. **生态根目录视图**分三区：顶部参谋长席位信息、中部框架真源+公告板只读卡片、底部 3 个子项目卡片。
> 4. **点击子项目卡片**切换到【子项目视图】，上部是双层时间轴（第一层项目迭代总览 + 第二层迭代详情），中部是 5 个角色卡片。
> 5. **双层时间轴**（子项目视图上部）：第一层「项目迭代总览轴」（横向，所有版本，数据源 INDEX.md 版本列表）；第二层「迭代详情轴」（点第一层节点后展示该迭代的 6 阶段流水线：PRD→设计→实现→部署检查→迭代关闭→收尾）。**进行中迭代展示阶段状态**（✅/🔵/⚪/🔴），**已完成迭代展示阶段产出内容摘要**（PRD是啥/设计是啥/实现是啥/部署是啥/关闭做了啥/收尾做了啥），不再显示规划。文档做轴（读 INDEX.md + 迭代记录），会话做钻取（点节点跳对话）。
> 6. **保留 v0.1 四视图不变**（工作台 / 部署 / 接入诊断 / 跨项目）。
> 7. **风格统一**：沿用 v0.1 的 shadcn/ui 体系，主色 `#030213`，圆角 10px，字重 ≤500。

## 1. 生态分层结构（重要）

```text
niuma-cheng 生态（ecosystem）
├── root-component（生态根维护，不单独加载）
│   ├── agent-workflow（框架真源）
│   └── coordination（公告板）
└── subproject（可选子项目，可配置角色会话）
    ├── 牛马程小报
    ├── AI 处理中枢
    └── 项目管理工作台（本项目）
```

**名称规则**：
- 项目名称以 `niuma-cheng-coordination/PROJECTS.md` 为真源，config 不自造名称
- workboard「项目管理工作台」是 v0.2 定位升级的新名，项目内先用；v0.2 迭代关闭检查时由参谋长在 coordination STATUS.md 元信息变更台账登记一行后同步真源

**配置结构**：
- `projects` 保持平铺数组 + `level` 字段，v0.1 四视图消费端零改动
- 新增 `ecosystem` 块承载参谋长席位/生态根信息
- 【项目会话】视图前端按 `level` 分组渲染

**关键规则**：
- 默认加载**生态根目录**，不是直接铺出所有子项目
- `agent-workflow`（kind=framework, level=root-component）和 `coordination`（kind=coordination, level=root-component）是**生态根维护的组成部分**，不作为可选子项目
- 真正的**可选子项目**（level=subproject）只有 3 个：xiaobao / ai / workboard

#### 3.2.2 第二层：迭代详情轴（6 阶段流水线）

**数据源**：`docs/progress/iterations/vX.Y.md`（阶段门禁状态） + 各阶段产出文档

**流水线 6 阶段**：

```
PRD → 设计 → 实现 → 部署检查 → 迭代关闭 → 收尾
```

**重要规则**：第二层根据迭代状态分两种展示模式——
- **进行中迭代**：展示阶段状态（✅/🔵/⚪/🔴），强调当前进度
- **已完成迭代**：展示阶段产出内容摘要（PRD是啥/设计是啥/实现是啥/部署是啥/关闭做了啥/收尾做了啥），强调做了什么

- **面包屑导航**：点击「生态根」返回生态根目录视图
- **角色卡片悬停**：悬停时显示阴影效果
- **角色卡片点击**：已映射角色点击打开详情抽屉；未选择角色点击打开配置弹窗
- **映射配置弹窗**：三步选择，选择项目后加载角色列表，选择角色后加载会话列表
- **确认配置**：点击确认后关闭弹窗，刷新角色卡片状态为「已映射」
- **取消配置**：点击取消后关闭弹窗，不保存配置

## 7. 状态覆盖

### 7.1 「项目会话」视图

| 状态 | 展示内容 |
|------|----------|
| 加载中 | 生态根目录视图或子项目视图显示骨架屏 |
| 空数据 | 无子项目配置时显示空态提示 |
| 正常状态 | 展示生态根目录视图或子项目视图 |

### 7.2 生态根目录视图

| 状态 | 展示内容 |
|------|----------|
| 参谋长席位已映射 | 绿色「已映射」徽标，显示会话摘要 |
| 参谋长席位未选择 | 灰色「未选择」徽标，可点击配置 |
| 框架真源/公告板卡片 | 只读展示，无操作按钮，无状态徽标 |
| 子项目卡片 | 显示项目名称、类型、角色映射概览，可点击切换 |

### 7.3 子项目视图

| 状态 | 展示内容 |
|------|----------|
| 已映射 | 绿色「已映射」徽标，显示会话信息摘要，可点击查看详情 |
| 未选择 | 灰色「未选择」徽标，可点击配置映射 |

### 7.4 映射配置弹窗

| 状态 | 展示内容 |
|------|----------|
| 初始状态 | 步骤 1（选择项目），角色和会话下拉禁用 |
| 选择项目后 | 步骤 2（选择角色），角色下拉启用 |
| 选择角色后 | 步骤 3（选择会话），会话下拉启用 |
| 配置完成后 | 确认按钮启用，可点击保存 |
