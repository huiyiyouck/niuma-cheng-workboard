## 0. 生态分层结构（v0.2 核心调整）

来源：`niuma-cheng/CLAUDE.md`（生态根参谋长入口） + `projects.config.json`

```text
niuma-cheng 生态（ecosystem）
├── root-component（生态根维护，不单独加载）
│   ├── agent-workflow（框架真源）
│   └── coordination（公告板）
└── subproject（可选子项目，可配置角色会话）
    ├── 牛马程小报
    ├── AI 处理中枢
    └── 项目管理工作台（本项目）
```

**名称规则**：
- 项目名称以 `niuma-cheng-coordination/PROJECTS.md` 为真源，config 不自造名称
- workboard「项目管理工作台」是 v0.2 定位升级的新名，项目内先用；v0.2 迭代关闭检查时必须在 coordination STATUS.md 元信息变更台账登记一行（名称/定位变更），由参谋长同步真源

**配置结构**：
- `projects` 保持平铺数组 + `level` 字段（方案 A），v0.1 四视图消费端零改动
- 新增 `ecosystem` 块承载参谋长席位/生态根信息
- 【项目会话】视图前端按 `level` 分组渲染即可

**关键规则**：
- 默认加载**生态根目录**，不是直接铺出所有子项目
- `agent-workflow`（kind=framework, level=root-component）和 `coordination`（kind=coordination, level=root-component）是**生态根维护的组成部分**，不作为可选子项目
- 真正的**可选子项目**（level=subproject）只有 3 个：xiaobao / ai / workboard
- v0.2 手动配置层级关系（`projects.config.json` 的 `level` 字段）
- v0.3+ 自动化预留（本次不实现）：项目元信息（名称/技术栈/状态）自动读取 `niuma-cheng-coordination/PROJECTS.md`；分层信息（level）无自动真源，保留 config 手动维护

## 0.1 kind 枚举表

| kind | 含义 | level | 第一版展示策略 |
|------|------|-------|----------------|
| `framework` | 框架真源，如 agent-workflow | `root-component` | 生态根目录视图中只读卡片展示，不进入子项目视图 |
| `coordination` | 跨项目协调仓库 | `root-component` | 生态根目录视图中只读卡片展示，不进入子项目视图 |
| `business-primary` | 业务主产品项目 | `subproject` | 子项目视图完整展示 5 角色 + 会话映射 |
| `business-service` | 业务服务方项目 | `subproject` | 子项目视图完整展示 5 角色 + 会话映射 |
| `tool` | 工具型项目（如 workboard） | `subproject` | 子项目视图完整展示 5 角色 + 会话映射 |

## 1. 用户流程

```
打开看板 → 默认【工作台】：项目卡片网格 + 待办汇总 + 跨项目协作横条
   │
   ├─ 左菜单新增【项目会话】 → 默认显示【生态根目录视图】
   │     │
   │     ├─ 生态根目录视图：
   │     │   - 顶部：参谋长席位信息（生态根会话状态）+ 配置按钮
   │     │   - 中部：框架真源 + 公告板 摘要卡片（只读，不可选）
   │     │   - 底部：项目组列表（3 个可选子项目卡片）
   │     │
   │     ├─ 点击子项目卡片 → 切换到【子项目视图】
   │     │   - 上部：双层时间轴
   │     │     · 第一层：项目迭代总览轴（横向，所有版本，点节点切换）
   │     │     · 第二层：迭代详情轴（6 阶段流水线）
   │     │       - 进行中迭代：展示阶段状态（✅/🔵/⚪/🔴）
   │     │       - 已完成迭代：展示阶段产出内容（PRD是啥/设计是啥/实现是啥...）
   │     │   - 中部：5 个角色卡片
   │     │   - 顶部有「返回生态根」按钮
   │     │   - 点击第二层节点 → 阶段详情抽屉 → 查看当时的对话 → 跳转对话视图
   │     │
   │     ├─ 参谋长席位（已映射）→ 点击席位卡 → 进入【对话视图】，展示参谋长会话的全部对话内容
   │     ├─ 角色卡片（已映射）→ 点击卡片 → 进入【对话视图】，展示该角色会话的全部对话内容
   │     ├─ 角色卡片（未映射）→ 显示「未选择」，可点击配置映射
   │     ├─ 点「配置映射」→ 配置弹窗：选择项目 → 选择角色 → 选择会话
   │     └─ 点参谋长席位「配置」→ 单步弹窗：选择会话
   │
   ├─ 【对话视图】：
   │   - 顶部：角色名称 + 返回按钮 + 会话状态
   │   - 中部：对话消息列表（按时间顺序展示用户和 Claude 的消息）
   │   - 底部：消息输入框（v0.2 只读展示，v0.3+ 支持输入）
   │
   ├─ 保留【工作台】/【部署】/【接入诊断】/【跨项目】视图
   └─ 其他流程同 v0.1
```

### 2.2 左菜单 = 5 个一级视图（按使用频率排序）

### 2.3 【项目会话】视图详情

#### 2.3.1 生态根目录视图（默认）

**顶部：参谋长席位信息区**
- 显示生态根目录名称：「niuma-cheng 生态」
- 显示参谋长席位状态（已映射/未选择）
- 显示生态根会话摘要
- **配置按钮**：席位卡右侧「配置」按钮（无论已映射/未选择状态均显示），点击后弹单步会话选择弹窗（项目定死生态根、角色定死参谋长），已映射状态下可重新配置换绑会话
- 复用子项目角色映射的同一套会话列表与存储机制

**中部：生态根维护的组成部分（只读卡片，不可选）**
- 框架真源卡片：`agent-workflow` 摘要（agent-workflow（框架真源））
- 公告板卡片：`coordination` 摘要（公告板（coordination））
- 这两个卡片只展示信息，**不能配置会话映射**，不变

**底部：项目组列表（可选子项目，level=subproject）**
- 展示 3 个子项目卡片：牛马程小报 / AI 处理中枢 / 项目管理工作台
- 每张卡片显示：项目名称、项目类型、角色映射概览（如「3/5 已映射」）
- 点击卡片 → 切换到【子项目视图】

#### 2.3.2 子项目视图

**顶部：面包屑导航**
- 「生态根」> [子项目名称]
- 点击「生态根」返回生态根目录视图

**上部：双层时间轴（文档做轴，会话做钻取）**

**第一层：项目迭代总览轴（横向）**
- 展示该项目的所有迭代版本，横向排列
- 数据源：`docs/progress/INDEX.md` 的「版本列表」表格
- 每个迭代节点显示：
  - 版本号（如 v0.1、v0.6.1）
  - 状态：✅ 已完成 / 🔵 进行中 / 📋 规划中
- 默认选中当前迭代（INDEX.md 的「当前迭代」字段）；无当前迭代时选中最新已完成迭代
- 横向滚动支持多迭代

**第二层：迭代详情轴（点击第一层节点后展示）**
- 展示选中迭代的 6 阶段流水线
- 数据源：`docs/progress/iterations/vX.Y.md`（阶段门禁状态） + 各阶段产出文档
- 流水线 6 阶段：PRD → 设计 → 实现 → 部署检查 → 迭代关闭 → 收尾

**进行中迭代（第二层展示阶段状态）**：
- 每个阶段节点显示：
  - 阶段名称
  - 状态：✅ 已定稿 / 🔵 进行中 / ⚪ 未开始 / 🔴 阻塞
  - 责任角色
  - 完成日期（已定稿的阶段）
- 钻取交互：
  - 点击已定稿阶段 → 阶段详情抽屉：产出文档链接 + 【查看当时的对话】按钮
  - 点击「查看当时的对话」→ 跳转对话视图
  - 进行中阶段 → 跳转到该角色的对话视图

**已完成迭代（第二层展示阶段产出内容）**：
- 每个阶段节点显示：
  - 阶段名称
  - 责任角色
  - 完成日期
  - 产出内容摘要（PRD 是什么、设计是什么、实现是什么、部署是什么、关闭做了什么、收尾做了什么）
- 钻取交互：
  - 点击阶段节点 → 阶段详情抽屉：产出文档链接 + 产出内容摘要 + 【查看当时的对话】按钮
  - 点击「查看当时的对话」→ 跳转对话视图

**中部：角色卡片区**
- 展示 5 个角色卡片：PM / Architect / Developer / DevOps / General
- 卡片布局：3 列网格
- 每张卡片内容：
  - 角色图标（lucide 图标）
  - 角色名称（中文）
  - 角色职责简述（读取 `docs/baseline/role-{role}.md` 的「我是谁」段落）
  - 映射状态徽标（已映射 / 未选择）
  - 操作按钮（配置映射 / 查看详情）

#### 2.3.3 映射状态徽标

| 状态 | 徽标文案 | 配色 |
|------|----------|------|
| 已映射 | 已映射 | primary / 绿 |
| 未选择 | 未选择 | muted 灰 |

#### 2.3.4 参谋长席位配置弹窗（单步）
- **触发**：点击参谋长席位卡右侧「配置」按钮
- **单步选择**：选择会话（下拉，从服务器读取的 Claude Code 历史对话记录）
- **项目定死**：生态根（niuma-cheng 生态）
- **角色定死**：参谋长
- **复用机制**：复用子项目角色映射的同一套会话列表与存储机制
- 底部按钮：确认配置 / 取消

#### 2.3.5 子项目角色映射配置弹窗（三步）
- **步骤 1**：选择项目（下拉，仅显示 3 个子项目：牛马程小报 / AI 处理中枢 / 项目管理工作台）
- **步骤 2**：选择角色（下拉，5 个角色列表）
- **步骤 3**：选择会话（下拉，从服务器读取的 Claude Code 历史对话记录）
- 底部按钮：确认配置 / 取消

#### 2.3.6 对话视图

**触发**：点击已映射的参谋长席位卡或角色卡片

**顶部栏**：
- 返回按钮：点击返回上一级（生态根目录视图 / 子项目视图）
- 角色/席位名称：当前会话的角色名称或参谋长席位
- 会话状态：在线 / 离线 / 最近活动时间

**中部：对话消息列表**：
- 按时间顺序展示（最新消息在底部）
- 消息类型区分：
  - 用户消息：左侧对齐，主色背景
  - Claude 消息：右侧对齐，次要色背景
- 消息内容：支持 markdown 渲染（代码块、列表、粗体等）
- 消息时间戳：每条消息下方显示发送时间
- 滚动条：自动滚动到底部，支持手动滚动查看历史

**底部：消息输入区域**（v0.2 只读展示，v0.3+ 支持输入）：
- 只读状态：显示「只读模式」提示，输入框禁用
- 提示文案：「v0.3+ 将支持直接在此输入消息」

#### 2.3.7 对话消息渲染规则

| 消息类型 | 对齐 | 背景色 | 边框 |
|----------|------|--------|------|
| 用户消息 | 左侧 | 主色背景 | 无 |
| Claude 消息 | 右侧 | 次要色背景（浅灰） | 无 |

**markdown 支持**：
- 代码块：带语言标识的代码块，语法高亮
- 列表：有序/无序列表
- 粗体/斜体
- 链接：可点击跳转
- 引用：灰色引用块

## 3. 关键交互状态

### 3.1 【项目会话】视图

- **加载中**：生态根目录视图或子项目视图显示骨架屏
- **空数据**：无子项目配置时显示空态提示
- **正常状态**：展示生态根目录视图或子项目视图

### 3.2 生态根目录视图

- **参谋长席位**：显示已映射/未选择状态
- **框架真源/公告板卡片**：只读展示，无操作按钮
- **子项目卡片**：显示角色映射概览，可点击切换

### 3.3 子项目视图

- **角色卡片（已映射）**：绿色「已映射」徽标，显示会话信息摘要，可点击查看详情
- **角色卡片（未选择）**：灰色「未选择」徽标，可点击配置映射
- **面包屑导航**：点击「生态根」返回上层

### 3.4 映射配置弹窗

- **步骤切换**：选择项目后加载角色列表，选择角色后加载会话列表
- **确认配置**：点击确认后，保存映射关系到数据库，关闭弹窗，刷新角色卡片状态
- **取消**：关闭弹窗，不保存配置

### 3.5 参谋长席位配置弹窗（单步）

- **初始状态**：会话下拉框直接启用，显示服务器读取的 Claude Code 历史对话记录列表
- **选择后状态**：选择会话后，确认按钮启用（之前为禁用状态）
- **确认配置**：点击确认后，保存映射关系（更新 `ecosystem.root_session_id`），关闭弹窗，刷新参谋长席位状态
- **取消**：关闭弹窗，不保存配置

## 4. 视觉约束

- **风格基线**：`docs/knowledge/ui/design-system.md`（shadcn/ui 浅色体系）
- **关键 token**：背景 `#f6f7f9` / 卡片白 / 主色 `#030213` / 圆角 10px / 字重 ≤500 / lucide 图标
- **状态徽标配色**：
  - 已映射：primary / 绿
  - 未选择：muted 灰
  - 读取异常：chart-4 橙

## 5. 验收标准（待 UI 确认后细化）

1. 当打开【项目会话】视图，应默认展示【生态根目录视图】。
2. 生态根目录视图应展示参谋长席位信息、框架真源/公告板摘要卡片、3 个子项目卡片。
3. 当点击子项目卡片，应切换到【子项目视图】，展示该项目的 5 个角色卡片。
4. 当点击面包屑「生态根」，应返回生态根目录视图。
5. 当角色已映射，应显示绿色「已映射」徽标和会话信息摘要。
6. 当角色未映射，应显示灰色「未选择」徽标和「配置映射」按钮。
7. 当点击「配置映射」按钮，应弹出配置弹窗，支持三步选择：项目 → 角色 → 会话。
8. 当完成映射配置并确认，应保存映射关系到数据库，角色卡片状态更新为「已映射」。
9. 当刷新页面，映射关系应保持不变。
10. v0.1 的四视图功能不受影响。
